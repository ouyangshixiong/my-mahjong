# BeiMi Mahjong AI Trainer

一个从旧贝密棋牌项目中拆出的麻将方向新子项目：

- `client/`：Electron 客户端，负责洗牌、发牌、摸打、牌河、四家单机回合、界面和交互。
- `server/`：轻量 Node.js 策略服务，提供玩法规则集、算番规则、向听/听牌/出牌建议，不保存牌局状态。
- `assets/`：复用旧项目中的房间背景与麻将语音资源。
- `tests/`：规则、AI 和服务 smoke 测试。

## 安装依赖

```bash
npm install
```

如果 Electron 二进制下载失败，可使用镜像重新安装：

```bash
ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/ npx install-electron --no
```

## 启动服务端

```bash
./start-server.sh
```

无参数时监听 `127.0.0.1:5057`。也可以显式指定监听地址和端口：

```bash
./start-server.sh 127.0.0.1 5057
```

部署到 VPS 并监听所有网卡：

```bash
./start-server.sh 0.0.0.0 5057
```

## 启动客户端

客户端只连接指定的服务端，不会在本机创建服务端进程：

```bash
./start-client.sh http://127.0.0.1:5057
```

连接 VPS 时传入 VPS 的实际地址，生产环境应使用 HTTPS 域名：

```bash
./start-client.sh https://mahjong-api.example.com
```

## 功能边界

当前版本是麻将训练/单机练习软件：

- 支持服务端下发玩法规则集，客户端可点击“更新玩法”重新拉取。
- 支持四川麻将血战玩法：三门 108 张，发牌后四家先定缺，再从各自定缺花色换出 3 张；缺门牌不足 3 张时必须全部选出并用其他牌补足，随后按顺时针/逆时针/对家随机方向同时交换。
- 定缺牌未打完时强制优先打缺门牌，清空后才可碰、杠或胡；四川玩法不可吃。
- 支持碰、直杠、暗杠、补杠、抢杠与杠后补牌；刮风下雨即时结算，流局未听会退回杠分。
- 支持点炮、一炮多响、最后四张强制胡；玩家胡牌后退出牌局，其余玩家继续到三家胡牌或牌墙摸完。
- 支持平胡、对对胡、清一色、七对、根、杠、金钩钓、杠上花、杠上炮、抢杠、海底等常用番型，默认 3 番封顶并维护四家积分。
- 血战流局执行退税、查花猪、查大叫，并按听牌者理论最大番结算。
- 支持红中玩法：三门加 4 张红中，红中作为癞子补顺子、刻子、对子。
- 支持四家摸打，玩家为东家。
- 支持标准 4 面子 1 将与七对子胡牌判断。
- 支持由服务端下发的算番表，并在客户端按当前玩法显示番型和总番。
- 支持 AI 出牌建议、听牌列表、估算向听、形状分。
- 服务端不处理支付、房卡、真钱、房间结算。

## 服务接口

- `GET /rulesets`：下发全部玩法和算番规则。
- `GET /rulesets/:id`：下发指定玩法。
- `POST /ai/analyze`：按 `rulesetId` 分析手牌。
- `POST /ai/exchange`：按起手牌和已选 `lackSuit` 返回换三张建议。
- `POST /ai/lack-suit`：按起手牌返回定缺建议。
- `POST /ai/discard`：按 `rulesetId` 返回 AI 出牌建议。
- `POST /ai/score`：按 `rulesetId` 算番。

`/ai/exchange`、`/ai/analyze`、`/ai/discard`、`/ai/score` 必须显式传入 `lackSuit`；四川血战传 `m`/`p`/`s`，无定缺玩法传 `null`。`/ai/discard` 还必须显式传入 `mustDiscard`：正常摸牌为 `false`，碰牌后必须出牌为 `true`。

## 规则依据与默认选项

胡牌、血战、杠分和流局结算参考以下四川麻将资料；本项目的房间流程明确配置为“定缺在前、换三张在后”，换牌花色固定为各自缺门：

- [Mahjong Lab：四川麻将规则](https://mjlab.app/docs/sichuan)
- [K神传说：四川麻将基本规则](https://zhuanlan.zhihu.com/p/22612168)
- [K神传说：四川麻将计番规则](https://zhuanlan.zhihu.com/p/22804668)

不同地区和平台对定缺/换牌顺序、自摸加底/加番、呼叫转移、封顶番数、花猪赔付对象等存在差异。本规则集默认先定缺再换牌、自摸加底、3 番封顶、花猪向未胡且非花猪者赔付、不开启呼叫转移；这些差异通过规则集字段显式配置，不在代码中静默降级。

下一阶段可以继续补：房间级规则选项、呼叫转移开关、更多成都番型、牌谱复盘、局后 AI 讲解。
