# BeiMi Mahjong AI Trainer

一个从旧贝密棋牌项目中拆出的麻将方向新子项目：

- `client/`：Electron 客户端，负责洗牌、发牌、摸打、牌河、四家单机回合、界面和交互。
- `server/`：轻量 Node.js 策略服务，提供玩法规则集、算番规则、向听/听牌/出牌建议，不保存牌局状态。
- `assets/`：复用旧项目中的房间背景与麻将语音资源。
- `tests/`：规则、AI 和服务 smoke 测试。

玩法参数以 `server/rulesets.js` 为唯一来源，通过 `/rulesets` 接口下发。客户端负责保存和推进单机牌局状态，可以执行服务端下发的规则，但不能在缺少字段时自行推断玩法或设置默认行为；未知规则值必须直接报错。

## 安装依赖

```bash
npm install
```

如果 Electron 二进制下载失败，可使用镜像重新安装：

```bash
ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/ npx install-electron --no
```

## 启动服务端

```bash
./start-server.sh
```

无参数时监听 `127.0.0.1:5057`。也可以显式指定监听地址和端口：

```bash
./start-server.sh 127.0.0.1 5057
```

部署到 VPS 并监听所有网卡：

```bash
./start-server.sh 0.0.0.0 5057
```

## 启动客户端

客户端只连接指定的服务端，不会在本机创建服务端进程：

```bash
./start-client.sh http://127.0.0.1:5057
```

连接 VPS 时传入 VPS 的实际地址，生产环境应使用 HTTPS 域名：

```bash
./start-client.sh https://mahjong-api.example.com
```

## 功能边界

当前版本是麻将训练/单机练习软件：

- 支持服务端下发玩法规则集，客户端可点击“更新玩法”重新拉取。
- 默认支持四川麻将血流成河玩法：三门 108 张，胡牌玩家不退出并可重复胡牌；胡牌后不能再碰，但遇到可杠的弃牌时仍由玩家选择“杠”或“过”；所有胡牌即时结算，直到牌墙摸完。
- 保留四川麻将血战玩法供开局前选择：已胡玩家退出，其余玩家继续到三家胡牌或牌墙摸完。
- 四川玩法发牌后四家先定缺，再从各自定缺花色换出 3 张；缺门牌不足 3 张时必须全部选出并用其他牌补足，随后按顺时针/逆时针/对家随机方向同时交换。
- 定缺牌未打完时强制优先打缺门牌，清空后才可碰、杠或胡；四川玩法不可吃。
- 支持碰、直杠、暗杠、补杠、抢杠与杠后补牌；所有杠牌提示和执行都必须保留当前胡牌听口；刮风下雨即时结算，流局未听会退回杠分。
- 四川牌序规则：碰、直杠会截断原牌序，操作完成后由碰杠者的下家摸牌；暗杠、补杠只在本人回合内进行，杠后补牌并出牌后仍按原顺序由其下家摸牌。
- 支持点炮、一炮多响、最后四张强制胡；一炮双响、三响会先播放专属牌桌语音，再依次播报各家胡牌番型；血流模式累计显示每家胡牌次数，血战模式显示头游、二游、三游流程。
- 支持平胡、对对胡、清一色、七对、根、金钩钓、杠上花、杠上炮、抢杠、海底等常用番型；每个已完成的杠在后续胡牌时增加 1 番，杠分仍即时单独结算；底分 100 分，默认 5 番封顶并维护四家积分。
- 点击任意玩家头像与“胡牌次数/分值”区域，可查看本局可滚动的分数增减明细。
- 四川玩法流局执行退税、查花猪、查大叫，并按听牌者理论最大番结算；参与者范围由服务端规则集的 `drawSettlementPlayerScope` 下发，默认排除已经胡牌的玩家。
- 支持红中玩法：三门加 4 张红中，红中作为癞子补顺子、刻子、对子。
- 支持四家摸打，玩家为东家，按“东家（你）→南家（下家）→西家（对家）→北家（上家）”逆时针行牌。
- 支持标准 4 面子 1 将与七对子胡牌判断。
- 支持由服务端下发的算番表，并在客户端按当前玩法显示番型和总番。
- 支持 AI 出牌建议、听牌列表、估算向听、形状分。
- 服务端不处理支付、房卡、真钱、房间结算。

## 麻将机器人算法

当前机器人使用的是本地规则引擎和确定性启发式评分算法，不是 AI 大模型，也不依赖 OpenAI、Claude、Gemini 或 Ollama 等外部推理服务。相同的手牌、明牌、定缺和玩法规则会产生相同的决策。

### 定缺

1. 分别假设万、筒、条为缺门。
2. 优先选择手中张数最少的花色。
3. 张数相同时，移除该花色后计算剩余手牌的结构分，选择结构分更高的方案。
4. 如果仍然同分，按万、筒、条的固定顺序决定，保证结果可重现。

### 换三张

- 当规则要求从定缺花色换牌时，只枚举该花色的合法三张组合。
- 如果缺门牌不足三张，选中全部缺门牌，再枚举其他花色的补足方案。
- 对每个候选组合模拟移除三张牌，计算剩余手牌的结构分，选择保留结构最好的方案。
- 四家确认后从顺时针、逆时针、对家三种方向中安全随机选择一次，并用桌面中央提示、方向箭头和四组三张牌的飞行动画展示同一次交换结果。

### 牌型结构分

结构分用于衡量对子、刻子、连张和嵌张等组牌潜力，当前权重如下：

| 手牌特征 | 分数 |
| --- | ---: |
| 同牌至少两张（对子） | `+18` |
| 同牌至少三张（刻子，叠加对子分） | `+35` |
| 同花色相邻连张 | `+13` |
| 同花色间隔一张的嵌张 | `+7` |
| 单张字牌 | `-6` |
| 单张幺九牌 | `-3` |
| 每张癞子牌 | `+20` |
| 四川玩法中每张尚未打出的缺门牌 | `-120` |

同一组牌可同时获得多项分数。例如刻子会同时获得对子分和刻子分。

### 胡牌、听牌和活张

- 胡牌判定支持标准的“4 面子 + 1 将”、七对以及规则集配置的癞子。
- 四川玩法中如果手里还有缺门牌，不允许胡牌。
- 听牌分析会枚举规则集允许的每种牌，模拟加入手牌后是否胡牌。
- 活张数按“规则中该牌总数 - 手牌中已知数量 - 桌面明牌数量”计算。

### 出牌决策

1. 如果当前手牌已胡且不是碰牌后必须出牌的状态，直接返回胡牌。
2. 枚举所有合法弃牌。定缺牌尚未清空时，候选范围只包含缺门牌。
3. 对每张候选牌模拟弃牌，计算弃牌后的结构分、听牌种类数和总活张数。
4. 使用以下公式排序：

   ```text
   出牌分 = 牌型结构分 + 听牌种类数 × 80 + 总活张数 × 35
   ```

5. 选择得分最高的弃牌；同分时按牌的固定排序决定。

### 碰、杠与决策优先级

当前电脑玩家的动作优先级是：

1. 其他玩家弃牌后，会同时判定当前玩家可用的胡、碰、杠操作并一次展示；选择“过”会放弃该张牌的全部操作。
2. 自己摸牌后，如果存在暗杠或补杠，会先选择第一个合法杠牌方案，再进入杠后补牌流程；最后四张强制胡的规则除外。
3. 没有可用自杠时，由出牌决策算法判定自摸胡或选择弃牌。
4. 其他玩家弃牌时，机器人仍按胡牌优先处理；无人胡牌时，能直杠则直杠，否则能碰则碰。
5. 碰牌后通过同一出牌评分算法选择要打出的牌。

### 向听数估算

- 已胡牌返回 `-1`，已听牌返回 `0`。
- 其他情况使用 `6 - floor(牌型结构分 / 42)` 进行估算，并将结果限制在 `1` 到 `6` 之间。
- 该值是用于训练提示的启发式估算，不是通过完整牌型搜索得到的严格向听数。

### 当前算法边界

- 出牌时只使用自己的手牌、已知明牌、定缺和规则集，不会推测对手暗牌。
- 没有对手建模、牌谱学习、神经网络、蒙特卡洛树搜索或自我对弈训练。
- 碰杠策略目前为固定优先级，不会比较“碰/杠/过”后的长期收益。
- 出牌评分主要偏向进攻和听牌速度，尚未计算危险牌、点炮概率、番数期望和牌局阶段。

## 服务接口

- `GET /rulesets`：下发默认玩法 ID、全部玩法和算番规则；默认玩法为 `sichuan-xueliu`。
- `GET /rulesets/:id`：下发指定玩法。
- `POST /ai/analyze`：按 `rulesetId` 分析手牌。
- `POST /ai/exchange`：按起手牌和已选 `lackSuit` 返回换三张建议。
- `POST /ai/lack-suit`：按起手牌返回定缺建议。
- `POST /ai/discard`：按 `rulesetId` 返回 AI 出牌建议。
- `POST /ai/score`：按 `rulesetId` 算番。

`/ai/exchange`、`/ai/analyze`、`/ai/discard`、`/ai/score` 必须显式传入 `lackSuit`；四川血战传 `m`/`p`/`s`，无定缺玩法传 `null`。`/ai/discard` 还必须显式传入 `mustDiscard`：正常摸牌为 `false`，碰牌后必须出牌为 `true`。

## 规则依据与默认选项

胡牌、血战、杠分和流局结算参考以下四川麻将资料；本项目的房间流程明确配置为“定缺在前、换三张在后”，换牌花色固定为各自缺门：

- [Mahjong Lab：四川麻将规则](https://mjlab.app/docs/sichuan)
- [K神传说：四川麻将基本规则](https://zhuanlan.zhihu.com/p/22612168)
- [K神传说：四川麻将计番规则](https://zhuanlan.zhihu.com/p/22804668)

座次与摸打顺序参考国标麻将的通用行牌规定：庄家为东风，下家为南风、对家为西风、上家为北风，并依座次逆时针抓牌和出牌。

- [国标麻将游戏官方网站：基本术语及一般规定](https://www.pook.com/mobile/gbmj/rule/rule-1.html)
- [国标麻将游戏官方网站：行牌规定](https://www.pook.com/mobile/gbmj/rule/rule-2.html)

不同地区和平台对定缺/换牌顺序、自摸加底/加番、呼叫转移、封顶番数、花猪赔付对象等存在差异。本规则集默认先定缺再换牌、普通自摸加 1 番、海底和杠上花替代普通自摸番、5 番封顶、流局结算只覆盖未胡玩家、花猪向未胡且非花猪者赔付、不开启呼叫转移；这些差异通过服务端规则集字段显式配置并下发，不在客户端自行推断或静默降级。
